# PRD-023: City Events & Exhibitions — Implementation Prompt

## Overview

Build the City Events feature for UB Trippin. This is our most important feature yet — a self-learning event discovery pipeline that curates exceptional exhibitions, festivals, and performances for travelers. Quality over quantity. We are the curator.

### Database (Supabase migrations)

Create migration files in `supabase/migrations/` with timestamps. **All tables need RLS policies.**

```sql
-- 1. tracked_cities
CREATE TABLE tracked_cities (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  city TEXT NOT NULL,
  country TEXT NOT NULL,
  country_code TEXT,
  slug TEXT UNIQUE NOT NULL, -- URL-safe: 'paris', 'new-york'
  latitude NUMERIC,
  longitude NUMERIC,
  timezone TEXT,
  hero_image_url TEXT,
  last_refreshed_at TIMESTAMPTZ,
  created_at TIMESTAMPTZ DEFAULT now(),
  UNIQUE(city, country)
);

-- RLS: public read, admin write
ALTER TABLE tracked_cities ENABLE ROW LEVEL SECURITY;
CREATE POLICY "Public read" ON tracked_cities FOR SELECT USING (true);

-- 2. tracked_venues
CREATE TABLE tracked_venues (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  city_id UUID REFERENCES tracked_cities(id) ON DELETE CASCADE,
  google_place_id TEXT UNIQUE,
  name TEXT NOT NULL,
  venue_type TEXT, -- museum, theater, concert_hall, gallery, stadium, outdoor, sacred_venue, club
  tier INTEGER DEFAULT 50, -- venue prestige 0-100
  photo_url TEXT,
  maps_url TEXT,
  added_by TEXT DEFAULT 'admin',
  created_at TIMESTAMPTZ DEFAULT now()
);

ALTER TABLE tracked_venues ENABLE ROW LEVEL SECURITY;
CREATE POLICY "Public read" ON tracked_venues FOR SELECT USING (true);

-- 3. city_events
CREATE TABLE city_events (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  city_id UUID REFERENCES tracked_cities(id) ON DELETE CASCADE,
  venue_id UUID REFERENCES tracked_venues(id),
  parent_event_id UUID REFERENCES city_events(id), -- festival lineup rollup
  title TEXT NOT NULL,
  venue_name TEXT,
  venue_type TEXT,
  category TEXT, -- art, music, theater, food, festival, sports, architecture, sacred, market
  event_tier TEXT NOT NULL DEFAULT 'medium', -- 'major', 'medium', 'local'
  description TEXT,
  start_date DATE NOT NULL,
  end_date DATE,
  time_info TEXT, -- "19:00" or "10:00-18:00"
  significance_score INTEGER DEFAULT 50,
  source TEXT,
  source_url TEXT,
  image_url TEXT,
  price_info TEXT,
  booking_url TEXT,
  tags TEXT[],
  lineup JSONB, -- for festivals: [{name, id, url}]
  last_verified_at TIMESTAMPTZ,
  expires_at DATE,
  created_at TIMESTAMPTZ DEFAULT now(),
  updated_at TIMESTAMPTZ DEFAULT now()
);

ALTER TABLE city_events ENABLE ROW LEVEL SECURITY;
CREATE POLICY "Public read" ON city_events FOR SELECT USING (true);
CREATE INDEX idx_city_events_city_dates ON city_events(city_id, start_date, end_date);
CREATE INDEX idx_city_events_tier ON city_events(event_tier);

-- 4. city_sources (per-city source list for the pipeline)
CREATE TABLE city_sources (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  city_id UUID REFERENCES tracked_cities(id) ON DELETE CASCADE,
  source_type TEXT, -- 'rss', 'tourism_board', 'facebook_page', 'newspaper', 'event_calendar', 'venue_site', 'api'
  name TEXT NOT NULL,
  url TEXT NOT NULL,
  language TEXT DEFAULT 'en',
  scrape_frequency TEXT DEFAULT 'weekly',
  status TEXT DEFAULT 'candidate', -- 'candidate', 'active', 'dormant', 'dead'
  consecutive_failures INTEGER DEFAULT 0,
  last_scraped_at TIMESTAMPTZ,
  last_event_count INTEGER DEFAULT 0,
  discovered_via TEXT, -- 'manual', 'ai_search', 'user_suggestion', 'rss_autodiscovery'
  notes TEXT,
  created_at TIMESTAMPTZ DEFAULT now()
);

ALTER TABLE city_sources ENABLE ROW LEVEL SECURITY;
CREATE POLICY "Public read" ON city_sources FOR SELECT USING (true);

-- 5. event_pipeline_diary (self-improvement)
CREATE TABLE event_pipeline_diary (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  city_id UUID REFERENCES tracked_cities(id) ON DELETE CASCADE,
  run_date DATE NOT NULL,
  diary_text TEXT NOT NULL,
  run_metadata JSONB NOT NULL DEFAULT '{}',
  next_day_plan JSONB,
  created_at TIMESTAMPTZ DEFAULT now(),
  UNIQUE(city_id, run_date)
);

ALTER TABLE event_pipeline_diary ENABLE ROW LEVEL SECURITY;
CREATE POLICY "Public read" ON event_pipeline_diary FOR SELECT USING (true);

-- 6. event_source_reports
CREATE TABLE event_source_reports (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  city_id UUID REFERENCES tracked_cities(id) ON DELETE CASCADE,
  source_name TEXT NOT NULL,
  source_url TEXT,
  run_date DATE NOT NULL,
  status TEXT NOT NULL,
  events_found INTEGER DEFAULT 0,
  error_message TEXT,
  duration_ms INTEGER,
  created_at TIMESTAMPTZ DEFAULT now()
);

ALTER TABLE event_source_reports ENABLE ROW LEVEL SECURITY;
CREATE POLICY "Public read" ON event_source_reports FOR SELECT USING (true);

-- 7. event_feedback (untrusted user input)
CREATE TABLE event_feedback (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES auth.users(id),
  city_id UUID REFERENCES tracked_cities(id),
  feedback_type TEXT NOT NULL,
  raw_text TEXT NOT NULL,
  sanitized_text TEXT,
  extracted_urls TEXT[],
  extracted_event JSONB,
  status TEXT DEFAULT 'pending',
  reviewed_by TEXT,
  review_notes TEXT,
  created_at TIMESTAMPTZ DEFAULT now()
);

ALTER TABLE event_feedback ENABLE ROW LEVEL SECURITY;
CREATE POLICY "Users can insert own feedback" ON event_feedback FOR INSERT WITH CHECK (auth.uid() = user_id);
CREATE POLICY "Users can read own feedback" ON event_feedback FOR SELECT USING (auth.uid() = user_id);

-- 8. user_event_interests (tracking clicks/saves)
CREATE TABLE user_event_interests (
  user_id UUID REFERENCES auth.users(id),
  event_id UUID REFERENCES city_events(id) ON DELETE CASCADE,
  action TEXT NOT NULL, -- 'saved', 'dismissed', 'clicked'
  created_at TIMESTAMPTZ DEFAULT now(),
  PRIMARY KEY(user_id, event_id)
);

ALTER TABLE user_event_interests ENABLE ROW LEVEL SECURITY;
CREATE POLICY "Users manage own interests" ON user_event_interests FOR ALL USING (auth.uid() = user_id);
```

### API Routes

#### `GET /api/v1/events?city=paris&from=2026-03-01&to=2026-03-07`
- Public API endpoint
- Returns events filtered by city slug, date range
- Optional: `tier=major`, `category=music`
- Rate limited

#### `POST /api/v1/events/feedback`
- Authenticated endpoint
- Accepts: city_id, feedback_type, text
- Sanitizes input (strip HTML, extract URLs, classify)
- Stores in event_feedback with status='pending'
- Rate limit: 3 per user per day

### Pro/Free Tier Logic

Follow the existing pattern in `src/lib/usage/limits.ts`:
- Use `getUserTier(userId, supabase)` to determine tier
- Free: show top 5-8 events, then fade-to-upsell overlay
- Pro: full page, all events, transparency section, feedback form
- Public city pages (no auth): show everything (for SEO), upsell on trip-specific pages

### Styling Rules

- Match existing UBT design: rounded-2xl, slate color palette, clean borders
- Indigo accent for events-specific elements (differentiates from trip items)
- Serif font (font-serif) for major event titles and city names
- Mobile-first: everything must work on 375px width
- No markdown tables in JSX (use flex/grid layouts)
- Dark mode: not required (UBT is light-only currently)

### Testing

Write tests in vitest for:
- Event tier rendering (major/medium/local show correct styles)
- Filter chip state management
- Upsell overlay shows for free users, hidden for pro
- API route validation (date parsing, rate limiting)
- Feedback sanitization (HTML stripping, URL extraction)

### What NOT to Do

- Do NOT use `createSecretClient()` or service role to bypass RLS. Fix RLS policies instead.
- Do NOT hardcode event data — all events come from the database
- Do NOT build the pipeline/cron (that's a separate phase) — build the frontend and API that reads from the tables
- Do NOT implement personalization — we are the curator, not the algorithm
- Do NOT add Spotify integration (deferred)
- Do NOT create admin UI for venues/cities (that's Phase 1b)
