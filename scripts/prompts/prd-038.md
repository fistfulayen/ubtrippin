# PRD-038: Weather-Aware Packing & Planning

## Your Mission

Implement the complete Weather-Aware Packing & Planning feature for UB Trippin as specified in the PRD below. Ship everything in ONE branch, ONE PR. All phases built together.

## Critical Rules

1. **Do NOT use createSecretClient() or service role to bypass RLS.** Fix RLS policies instead. This code will be security-audited monthly.
2. **Build unit tests** for every service module. Run them after each phase. Run them all at the end.
3. **Test the build** (`pnpm build`) after major changes. Don't let errors accumulate.
4. **Address ALL code review findings** before calling the PR complete. Read the Gemini and Claude review comments on the PR and fix every issue.
5. **Packing suggestions should be FUN and WITTY.** Not bland corporate suggestions. Give them personality. The AI prompt for packing generation should encourage humor.
6. **Share page viewers see weather but NOT packing suggestions.** Family collaborators see everything. Check `share_enabled` vs collaborator status.

## Branch

Create branch: `feature/prd-038-weather`

## What to Build

### Backend (src/lib/weather/)
- `geocode.ts` — Resolve city names to lat/lng via Open-Meteo geocoding API. Cache geocoding results.
- `forecast.ts` — Fetch forecasts from Open-Meteo. 16-day window only. If trip is beyond 16 days, return nothing (no averages, no placeholders).
- `cities.ts` — Extract cities from trip items. **Prioritize hotel locations over airport/flight locations.** Filter out layovers under 6 hours. Deduplicate. Order chronologically.
- `packing.ts` — AI-generated packing list from weather data. Use the same AI pattern as email extraction. Make suggestions funny and personality-driven.
- `weather-icons.ts` — Map Open-Meteo weathercode to emoji icons.

### Database Migration
- Create `trip_weather_cache` table with RLS (see PRD for full SQL).
- RLS: users see/modify only their own weather data.

### API Routes
- `src/app/api/v1/trips/[id]/weather/route.ts` — Public API (API key auth)
- `src/app/api/trips/[id]/weather/route.ts` — Internal web app route (Supabase auth)
- GET returns weather + packing for a trip. POST /refresh force-refreshes.
- Cache: 6 hours stale window. Bust on trip item changes.

### UI Components (src/components/trips/weather/)
- `weather-section.tsx` — Main container, renders below trip timeline
- `temp-range-bar.tsx` — Horizontal gradient showing min→max temp across trip
- `city-weather-card.tsx` — Per-city card with icon, temps, precip
- `packing-suggestions.tsx` — Collapsible AI packing list by category

### UI Integration
- Add WeatherSection to trip detail page (`src/app/(dashboard)/trips/[id]/page.tsx`), below the timeline
- **PRO gate:** Weather cards visible to ALL users. Packing suggestions visible only to PRO users (blurred + "Upgrade to PRO" CTA for free users).
- Responsive: horizontal scroll on mobile, grid on desktop
- Loading: skeleton loaders
- Empty states: "Add flights or hotels to see weather" when no locations
- Trip > 16 days: section hidden entirely

### Settings
- Add Celsius/Fahrenheit toggle to user settings page
- Store preference in user profile or local storage

### Share Page
- Weather cards visible on share page (public viewers)
- Packing suggestions HIDDEN on share page (the viewer isn't traveling)
- Family collaborators (authenticated, part of the trip) see packing suggestions

### CLI
- Add `weather` subcommand to `ubt trips` in `cli/ubt`
- Calls GET /api/v1/trips/:id/weather
- Flags: --json, --refresh, --celsius
- Free users: temps only. PRO users: full output + packing.

### MCP Tool
- Register `get_trip_weather` tool in existing MCP setup
- Same payload as API endpoint

### Tests
- Unit tests for: geocode, forecast, cities (hotel priority, layover filtering, dedup), packing prompt, cache logic
- Component tests where practical
- Run `pnpm test` and `pnpm build` before finishing

## Open-Meteo API Reference

Forecast: `https://api.open-meteo.com/v1/forecast`
Parameters: `latitude`, `longitude`, `daily=temperature_2m_max,temperature_2m_min,precipitation_sum,precipitation_probability_max,weathercode,wind_speed_10m_max`, `temperature_unit=fahrenheit`, `timezone=auto`

Geocoding: `https://geocoding-api.open-meteo.com/v1/search?name={city}&count=1`
