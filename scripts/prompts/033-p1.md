### 2A. Fix auth_rls_initplan (55 policies)

Every RLS policy using bare `auth.uid()` or `auth.jwt()` must be wrapped in a subselect.

**Before:** `USING (user_id = auth.uid())`  
**After:** `USING (user_id = (SELECT auth.uid()))`

This prevents per-row function re-evaluation. The subselect is evaluated once per query.

**Migration file:** `supabase/migrations/20260304000002_rls_initplan_fix.sql`

Query the current policies to get their exact expressions:
```sql
SELECT c.relname, pol.polname, pol.polcmd,
  pg_get_expr(pol.polqual, pol.polrelid) as using_expr,
  pg_get_expr(pol.polwithcheck, pol.polrelid) as check_expr
FROM pg_policy pol
JOIN pg_class c ON pol.polrelid = c.oid
JOIN pg_namespace n ON c.relnamespace = n.oid
WHERE n.nspname = 'public'
ORDER BY c.relname, pol.polname;
```

For each policy that contains `auth.uid()` or `auth.jwt()` without a wrapping `(SELECT ...)`:
1. `DROP POLICY IF EXISTS "<name>" ON public.<table>;`
2. Recreate with the same logic but all `auth.uid()` → `(SELECT auth.uid())` and `auth.jwt()` → `(SELECT auth.jwt())`

**IMPORTANT:** Preserve the exact same access logic. Only change the auth function wrapping. The `is_trip_owner()`, `is_family_member()`, `is_family_admin()`, `is_family_member_of()` helper functions internally call `auth.uid()` — those need to be updated too:
- Query their definitions with `pg_get_functiondef(oid)`
- Recreate them with `(SELECT auth.uid())` inside
- Then recreate the policies that use them

### 2B. Consolidate multiple permissive policies (70 warnings)

For each table that has multiple PERMISSIVE policies for the same role+action, merge them into a single policy with OR conditions.

**Migration file:** `supabase/migrations/20260304000003_consolidate_rls_policies.sql`

Example for `trips` SELECT (currently 3 separate permissive SELECT policies):
```sql
-- Drop the separate ones
DROP POLICY IF EXISTS "Users can manage own trips" ON public.trips;  -- FOR ALL (covers SELECT)
DROP POLICY IF EXISTS "trips_select" ON public.trips;                -- FOR SELECT (owner + collaborator)
DROP POLICY IF EXISTS "trips_family_read" ON public.trips;           -- FOR SELECT (family)

-- Recreate as consolidated policies (separate SELECT from write ops)
CREATE POLICY "trips_select" ON public.trips FOR SELECT USING (
  user_id = (SELECT auth.uid())
  OR EXISTS (SELECT 1 FROM trip_collaborators tc WHERE tc.trip_id = trips.id AND tc.user_id = (SELECT auth.uid()) AND tc.accepted_at IS NOT NULL)
  OR is_family_member(user_id)
);

CREATE POLICY "trips_insert" ON public.trips FOR INSERT WITH CHECK (
  user_id = (SELECT auth.uid())
);

CREATE POLICY "trips_update" ON public.trips FOR UPDATE USING (
  user_id = (SELECT auth.uid())
  OR EXISTS (SELECT 1 FROM trip_collaborators tc WHERE tc.trip_id = trips.id AND tc.user_id = (SELECT auth.uid()) AND tc.role = 'editor' AND tc.accepted_at IS NOT NULL)
  OR is_family_member(user_id)
) WITH CHECK (
  user_id = trip_owner_user_id(id)
  AND (
    user_id = (SELECT auth.uid())
    OR EXISTS (SELECT 1 FROM trip_collaborators tc WHERE tc.trip_id = trips.id AND tc.user_id = (SELECT auth.uid()) AND tc.role = 'editor' AND tc.accepted_at IS NOT NULL)
    OR is_family_member(user_id)
  )
);

CREATE POLICY "trips_delete" ON public.trips FOR DELETE USING (
  user_id = (SELECT auth.uid())
);
```

Do this for EVERY table with multiple permissive policies. Break the `FOR ALL` policies into separate per-action policies so the consolidation is clean.

### 5A. Fix local time preservation at insertion

**File:** `src/lib/utils.ts` — Add utility:
```typescript
/**
 * Extract local time (HH:MM) from an ISO 8601 timestamp.
 * The time portion of the ISO string represents local time when an offset is present.
 * e.g., "2026-03-10T12:52:00-05:00" → "12:52"
 */
export function extractLocalTime(isoString: string | null | undefined): string | null {
  if (!isoString) return null
  const match = isoString.match(/T(\d{2}:\d{2})/)
  return match ? match[1] : null
}
```
