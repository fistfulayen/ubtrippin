## Bug 2: Unable to edit guide city/country name

**Problem:** The PATCH API at `/api/v1/guides/:id` already supports updating `city`, `country`, and `country_code`, but the dashboard UI (`src/app/(dashboard)/guides/[id]/page.tsx`) has no edit control. Users must delete and recreate guides to fix a typo.

**Fix:** 
1. Add a server action `updateGuideMetadata` in `src/app/(dashboard)/guides/actions.ts`:
   - Takes `guideId`, `city`, `country`, `country_code`
   - Uses the standard `createClient()` from `@/lib/supabase/server` (NOT createSecretClient)
   - Updates via supabase `.update()` with `.eq('user_id', user.id)` for RLS
   - Calls `revalidatePath(`/guides/${guideId}`)`

2. Create a new client component `src/app/(dashboard)/guides/[id]/guide-edit-title.tsx`:
   - Shows current city name as the h1 title
   - Clicking it (or a small pencil icon) enters edit mode with an inline text input
   - On save, calls the server action
   - Also allow editing the country via a dropdown (reuse or import the CountrySelect pattern)
   - Keep it simple: inline edit for city name, small edit icon trigger

3. Update `src/app/(dashboard)/guides/[id]/page.tsx` to use the new `GuideEditTitle` component in place of the static h1.

## Bug 3: Guide detail page mobile layout issues

**Problem:** On mobile viewports, the header action buttons (Public, Copy link, Markdown, Add place, Cover image, Delete) overflow and stack awkwardly. The share URL at the bottom overflows its container horizontally.

**Fix in `src/app/(dashboard)/guides/[id]/page.tsx`:**

1. **Header buttons responsive layout:** The `flex items-center gap-2 flex-wrap justify-end` div containing the buttons needs better mobile handling. Wrap the secondary actions (Markdown export, Cover image, Delete) in a dropdown menu on small screens. Use a simple approach:
   - On mobile (sm:), show only the primary actions (Share toggle, Add place) inline
   - Group Markdown, Cover image, Delete into a "..." overflow menu using a Popover or simple dropdown
   - Use `hidden sm:flex` / `flex sm:hidden` pattern for the responsive switch

2. **Share URL overflow:** In the share info section at the bottom, add `break-all` to the URL link:
   ```tsx
   className="underline hover:no-underline font-mono text-xs break-all"
   ```
