There's a Japanese concept — *tsukumogami* — the idea that objects acquire souls after a hundred years. We're trying to do it in a hundred days.

## What We Built

This was a Monday morning. Eight pull requests merged before the founder left for the mountain.

Margot booked two hotels in Japan through her own account. They appeared in a separate trip instead of landing in the shared Golden Week itinerary. The family trip matcher required exact city-name matching — "Takaragawa Onsen Ousenkaku" does not equal "Tokyo." We looked at this for a while and decided: that's correct. If two family members travel to different cities on the same dates, you don't want their trips merged. The system was working as designed.

The real bug was subtler: the auto-created trip had no cover image. Unsplash returned zero results for "Takaragawa Onsen Ousenkaku" — too obscure. The fix: when the specific location fails, fall back to the AI-generated trip title. "Japan Onsen Hot Springs Getaway" returns forty beautiful landscape photos. The machine that names your trip now also helps illustrate it.

Hotel bookings from providers like Booking.com showed the provider name as the title instead of the hotel name. Your itinerary would say "Booking.com" four times instead of the names of the places you're actually staying. Fixed — the hotel name now leads, with the provider and confirmation code underneath.

We dogfooded the CLI by adding a hotel manually — a sake brewery guesthouse in Hokuto that was booked by a friend with no email confirmation. The CLI crashed four times. Short IDs rejected. Python tracebacks. Missing config. So we wrote a PRD, spun up a coding agent, and shipped the fix. Three commands now. Zero crashes.

Every PR gets reviewed by two AI models before a human sees it. This week they caught: Python code injection in a shell script, JSON injection that could escalate invite permissions, unauthenticated API calls leaking data shape, and XSS via unsanitized markdown. All fixed before merge. The paranoia is automated. The code is open source.

We shipped a feedback system, then discovered it was already there but disabled. Two user-submitted bugs — "can't find the privacy policy" and "inbox looks broken on mobile" — both fixed and marked shipped within the hour.

And then we built this. The page you're reading. A dispatches section at `/dispatches`, a weekly rhythm, an RSS feed. Markdown files in the repo, no CMS, no database. Just the words.

## The Numbers

204 commits this week. 8 PRs in one session. 30 PRDs written since inception, of which 26 are shipped. 14 users.

The activation rate hasn't moved. Still 17%. We know this is the problem. We're not hiding from it.

## What We Learned

When your deployment platform goes down, check their status page before rewriting your initialization code. We spent thirty minutes converting a Stripe client from module-level to lazy-loaded to proxy-wrapped to function-called, reverting each attempt, before discovering it was a platform-wide outage. The build was fine. The deploy step was broken for everyone.

The meta-lesson: when something breaks and you can't explain why, the explanation might not be in your code.

## What's Next

The CLI is now agent-friendly. The blog exists. The feedback loop is automated. Next week we write the first real dispatch — this is the backstory, the catching up, the "previously on."

Going forward: one dispatch every Sunday. What we built, what broke, the numbers, what we learned. No spin. This is how we find our people.
